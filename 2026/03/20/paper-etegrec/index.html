ETEGREC

1.Introduction

  • 问题：
    现有生成式推荐，SID生成与产生推荐结果两个阶段完全解耦，造成以下缺点：

    • item tokenizer感受不到推荐效果、也感受不到推荐要优化的目标。
    • 后面的recommender看不到最开始item的上下文信息，只能看到物品最后SID。
      后面的推荐相关训练不能用来重新优化前面学习到的SID，无法精炼item SID。

  • 工作：
    提出双编码器-解码器框架，可以将项目标记器和生成推荐器联合优化。
    设计了面向推荐的两种校准方法：序列-项目校准、偏好-语义校准。

    目的：无缝整合标记器和推荐器，促进二者协同增强。

2.METHOD

2.1 形式化问题

项目标记化：把每一个item映射为多个tokens的过程

输入交互序列通过item tokenization之后得到标记序列：X=[c11,c21,...,cL−1t,cLt]X=[c_1^1,c_2^1,...,c_{L-1}^t,c_L^t]X=[c11​,c21​,...,cL−1t​,cLt​]。
生成式推荐问题可以形式化为典型序列到序列学习问题：

2.2.2 生成推荐过程

采用类似于T5的Transformer编码器-解码器架构进行序列行为建模。

  • 编码过程：

    • 项目级用户交互序列SSS和目标项目it+1i_{t+1}it+1​首先被tokenized为标记序列X=[c11,c21,...,cLt]X=[c_1^1,c_2^1,...,c_L^t]X=[c11​,c21​,...,cLt​]和Y=[c1t+1,...,cLt+1]Y=[c_1^{t+1},...,c_L^{t+1}]Y=[c1t+1​,...,cLt+1​]。

    • 对应标记嵌入EX∈R∣X∣×dhE^X\in\mathbb{R}^{|X|\times d_h}EX∈R∣X∣×dh​（dhd_hdh​为推荐器隐藏大小）输入整个架构的第二部分-生成推荐器进行用户偏好建模。

    • 为什么EXE^XEX维度是∣X∣×dh{|X|\times d_h}∣X∣×dh​，X的长度为∣X∣|X|∣X∣，每一个对应的code，都被转换为对应的dhd_hdh​维度的code embedding，所以EXE^XEX维度是∣X∣×dh{|X|\times d_h}∣X∣×dh​。

    • 标记序列encoding的过程：

      HE=EncoderR(EX),(11) \boldsymbol{H}^E=\mathrm{Encoder}_R(\boldsymbol{E}^X),\tag{11} HE=EncoderR​(EX),(11)

      其中HE∈R∣X∣×dh\boldsymbol{H}^E\in\mathbb{R}^{|X|\times d_h}HE∈R∣X∣×dh​为encode过程之后得到的序列表示。
      此时HE\boldsymbol{H}^EHE里面的每一个向量，已经整合好了序列的上下文信息。

    • **Transformer编码器的一个重要特征：输入和输出的形状保持一致。**都是dhd_hdh​维。

  • 解码过程：

    • 原本Y=[c1t+1,...,cLt+1]Y=[c_1^{t+1},...,c_L^{t+1}]Y=[c1t+1​,...,cLt+1​]，构建新的解码器输入：Y~=[[BOS],c1t+1,...,cLt+1]\tilde{Y}=[[\mathrm{BOS}],c_1^{t+1},...,c_L^{t+1}]Y~=[[BOS],c1t+1​,...,cLt+1​]。添加BOS是Transformer架构自回归必须的一步。
      将HE\boldsymbol{H}^EHE与Y~\tilde{Y}Y~输入解码器提取用户偏好表示：

      HD=DecoderR(HE,Y~),(12) \boldsymbol{H}^D=\mathrm{Decoder}_R(\boldsymbol{H}^E,\tilde{Y}),\tag{12} HD=DecoderR​(HE,Y~),(12)

      其中HD∈R(L+1)×dh\boldsymbol{H}^D\in\mathbb{R}^{(L+1)\times d_h}HD∈R(L+1)×dh​为解码器隐藏状态，隐含用户对项目的偏好。

    • 解码过程研究：
      将Y~\tilde YY~一次性输入给解码器，对于第x个位置的预测，就不需要x-1位置的结果必须出来。

      我们将长度为 L+1L+1L+1 的 Y~\tilde{Y}Y~ 一次性输入了解码器，并且并行完成了计算。Transformer 保持输入输出长度一致的特性，直接“吐出”了包含 L+1L+1L+1 个向量的矩阵：

      HD=DecoderR(HE,Y~)∈R(L+1)×dh \boldsymbol{H}^D=\mathrm{Decoder}_R(\boldsymbol{H}^E,\tilde{Y}) \in \mathbb{R}^{(L+1)\times d_h} HD=DecoderR​(HE,Y~)∈R(L+1)×dh​

      研究HD\boldsymbol{H}^DHD 矩阵的内部：

      HD\boldsymbol{H}^DHD 的第 0 行向量 h0D\boldsymbol{h}_0^Dh0D​：仅仅结合了历史 HE\boldsymbol{H}^EHE 和 `[BOS]` 算出来的偏好。**（准备去预测 c1c_1c1​）** HD\boldsymbol{H}^DHD 的第 1 行向量 h1D\boldsymbol{h}_1^Dh1D​：结合了历史 HE\boldsymbol{H}^EHE、`[BOS]` 和正确的 c1c_1c1​ 算出来的偏好。**（准备去预测 c2c_2c2​）**

      …

      HD\boldsymbol{H}^DHD 的第 L 行向量 hLD\boldsymbol{h}_L^DhLD​：结合了历史 HE\boldsymbol{H}^EHE 和前面所有的正确 token 算出来的偏好。**（准备去预测结束符或后续行为）**

    • 训练时，可以一步直接得到结果。
      推理时，每一步都必须等待之前结束再计算，一步一步来计算最后结果。

    • 优势分析：(将Y~\tilde YY~一次性输入给解码器)
      首先是迅速，因为可以并行处理每一个位置的结果。（因为掩码机制存在，这样并没有作弊问题）
      其次是高效：对于每一个位置，都进行了训练。
      如果我们真的使用前一步预测的结果来进行下一步的预测，第一步猜错时，后面就会一步错、步步错。训练效率很低。
      这里的高效，可以表示为：Teacher Forcing（教师强制）避免误差累积。

  • 推荐损失：
    基于序列到序列范式优化目标标记的负对数似然：

    LREC=−∑j=1Llog⁡P(Yj∣X,Y<j),(13) \mathcal{L}_{\mathrm{REC}}=-\sum_{j=1}^L\log P(Y_j|X,Y_{<j}),\tag{13} LREC​=−j=1∑L​logP(Yj​∣X,Y<j​),(13)

    其中YjY_jYj​为目标标记第jjj个标记，Y<jY_{<j}Y<j​为YjY_jYj​前所有标记。通过此方式自回归生成目标项目标记。

2.3 面向推荐的校准

2.3.1序列-项目校准SIA

  • 假设：
    一个物品的编码器隐藏状态HE\boldsymbol H ^EHE与协同嵌入zzz高度相关。

    zzz是协同语义嵌入。 HE\boldsymbol H^EHE是 `token级别的用户历史交互序列信息`对应的嵌入EX\boldsymbol{E}^XEX经过encode之后得到的结果。

  • 1.压缩序列信息

    zE=MLP(mean_pool(HE)),(14) \boldsymbol{z}^E=\mathrm{MLP}(\mathrm{mean\_pool}(\boldsymbol{H}^E)),\tag{14} zE=MLP(mean_pool(HE)),(14)

    HE\boldsymbol{H}^EHE：Transformer 编码器的输出，它是一个矩阵（长度 × 维度），包含了序列中每个位置的信息。mean_pool\mathrm{mean\_pool}mean_pool：取平均值，把整个序列的信息压缩成一个向量。
    MLP\mathrm{MLP}MLP：一个简单的神经网络层，负责把推荐系统的“偏好空间”映射到分词器的“语义空间”。
    结果：zE\boldsymbol{z}^EzE 就代表了从“用户历史”中提取出来的、对下一个商品的预测特征。

  • 2.获取概率分布
    根据公式6.
    得到下面两个概率分布：
    PzlP_z^lPzl​：把目标商品 zzz 丢进 RQ-VAE，在第 lll 层码本（Codebook）上，它属于每一个 Token 的概率分布。
    PzElP_{z^E}^lPzEl​：把用户序列特征 zEz^EzE 丢进同一个 RQ-VAE，它认为下一时刻应该产生的 Token 概率分布。

  • 3.强制分布一致：

    LSIA=−∑l=1L(DKL(Pzl∣∣PzEl)+DKL(PzEl∣∣Pzl))\mathcal{L}_{\mathrm{SIA}}=-\sum_{l=1}^L\left(D_{KL}(P_z^l||P_{z^E}^l)+D_{KL}(P_{z^E}^l||P_z^l)\right)LSIA​=−l=1∑L​(DKL​(Pzl​∣∣PzEl​)+DKL​(PzEl​∣∣Pzl​))

    KL 散度：是衡量两个概率分布有多不像的指标。值越大，分布差异越大。
    对称性：同时计算 A→BA \rightarrow BA→B 和 B→AB \rightarrow AB→A 的散度，让对齐更稳健。
    含义：这个损失函数要求，对于 RQ-VAE 的每一层（111 到 LLL 层），从“用户历史”预测出来的 Token 分布，必须和“真实商品”生成的 Token 分布保持同步。

2.3.2 偏好-语义校准

  • 参与校准的两部分：
    要在连续的向量空间里做对齐，我们需要找到代表“用户”的向量和代表“商品”的向量：

    • 1.用户的宏观偏好 hD\boldsymbol{h}^DhD
      解码器输出矩阵 HD\boldsymbol{H}^DHD 的第一列
      Transformer 解码器在输入 [BOS][BOS][BOS] 符号后，输出的第一个隐藏状态向量。
      为什么使用第一列：
      此时，解码器还没开始预测任何具体的 Token，它完全凭借编码器传来的历史信息 HE\boldsymbol{H}^EHE 酝酿出了对下一个商品的整体预期。所以，这个 hD\boldsymbol{h}^DhD 包含了最纯粹的、未被具体 Token 细节干扰的“用户整体偏好”。

      这里的偏好是decoder在收到bos之后的输出，应该是矩阵里面第一行，paper里面一直用列，自己知道就好

    • 2.目标商品的重建语义z~\tilde zz~：
      是RQVAE从物品提取出token之后，再试图通过解码器还原回去得到的向量。

  • 校准假设：
    用户心里的整体预期（hD\boldsymbol{h}^DhD） 和 **这件商品的内在灵魂（z~\tilde{z}z~）**应该是有联系的。

  • 对齐过程：(采用对比学习里经典的infoNCE损失)

    LPSA=−(log⁡exp⁡(s(z~,hD)/τ)∑h^∈Bexp⁡(s(z~,h^)/τ)+log⁡exp⁡(s(hD,z~)/τ)∑z^∈Bexp⁡(s(hD,z^)/τ)),(16) \mathcal{L}_{\mathrm{PSA}}=-\left(\log\frac{\exp(s(\tilde{z},\boldsymbol{h}^D)/\tau)}{\sum_{\hat{\boldsymbol{h}}\in\mathcal{B}}\exp(s(\tilde{z},\hat{\boldsymbol{h}})/\tau)}+\log\frac{\exp(s(\boldsymbol{h}^D,\tilde{z})/\tau)}{\sum_{\hat{z}\in\mathcal{B}}\exp(s(\boldsymbol{h}^D,\hat{z})/\tau)}\right),\tag{16} LPSA​=−(log∑h^∈B​exp(s(z~,h^)/τ)exp(s(z~,hD)/τ)​+log∑z^∈B​exp(s(hD,z^)/τ)exp(s(hD,z~)/τ)​),(16)
    • log⁡exp⁡(s(z~,hD)/τ)∑h^∈Bexp⁡(s(z~,h^)/τ)\log\frac{\exp(s(\tilde{z},\boldsymbol{h}^D)/\tau)}{\sum_{\hat{\boldsymbol{h}}\in\mathcal{B}}\exp(s(\tilde{z},\hat{\boldsymbol{h}})/\tau)}log∑h^∈B​exp(s(z~,h^)/τ)exp(s(z~,hD)/τ)​:

      分子s(z~,hD)s(\tilde{z},\boldsymbol{h}^D)s(z~,hD) ：
      计算目标商品 z~\tilde{z}z~ 和当前用户偏好 hD\boldsymbol{h}^DhD 的余弦相似度。我们希望这个值越大越好。
      分母：

      B\mathcal{B}B 代表当前训练的这一个批次（Batch）里的所有其他用户（Batch Size 是256，就会有255个其他用户的偏好 h^\hat{\boldsymbol{h}}h^）。分母把目标商品和**所有用户**（包括真用户和 255 个路人）的相似度加起来。

      意义：
      在当前批次的一堆用户中，目标商品 z~\tilde{z}z~ 必须觉得当前的 hD\boldsymbol{h}^DhD 是最匹配的，对其他人的偏好不理不睬。

    • 损失里面的第二部分的解释与上面类似。

3.总结

​ 过去Tiger的范式，都把id生成和推荐结果两个阶段解耦，作者认为这样不容易让两部分深度融合，希望可以根据后面的推荐结果的好坏来训练到第一阶段的模型，也就是实现端到端的统一框架。

​ 作者设计两种对齐，序列-项目校准与偏好-语义校准。第一个对齐：1>用户交互历史token化之后再通过encoder得到的编码器隐藏状态HE\boldsymbol H ^EHE 2>物品的协同嵌入zzz。1代表序列、2代表项目。第二个对齐：3>第二阶段的解码器输入[BOS]之后得到的输出hD\boldsymbol{h}^DhD 4>物品的重建语义嵌入z~\tilde zz~。3代表用户的偏好、4代表物品的语义。

​ 通过上述两种对齐，设置对应损失函数，训练中，二阶段的推荐相关损失可以直接传播到(影响)一阶段的ID生成，两部分不再解耦，实现端到端。

  • tiger和letter里面，id生成更像是一个预处理步骤，这里的etegrec就是把两部分联合起来，实现端到端的统一框架。
